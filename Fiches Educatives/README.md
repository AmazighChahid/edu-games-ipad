# Tour de Hanoï — Dossier de spécification (pour Claude Code)

Ce dossier sert d’instructions **produit + pédagogie + UX/UI** pour implémenter l’activité **Tour de Hanoï** dans une application iPad (Expo / React Native) destinée aux enfants de **6 à 10 ans** (focus **7 ans**).

Objectif : fournir un brief **actionnable** pour coder l’expérience, sans “roman pédagogique”, tout en conservant les éléments essentiels Montessori (auto-correction, autonomie, guidance douce) et les mécanismes UI enfant (tactile, pictos, texte minimal).

## Contenu du dossier

- `01-overview.md` — Résumé activité + objectifs + contraintes
- `02-ux-flow.md` — Parcours écran par écran + états + règles d’adaptation
- `03-ui-spec.md` — Spécification UI iPad + composants + tailles + layout
- `04-feedback-ai.md` — Feedback, erreurs, indices, IA copilote (scripts courts)
- `05-parent-space.md` — Espace parent (données, copy, limites)
- `06-assets-sound.md` — Micro-animations, sons, voix (optionnel)

## Règles de rédaction (pour implémentation)

- **Texte enfant** : phrases ≤ 10 mots, vocabulaire concret, 1 idée par message.
- **Aide** : “question → indice 1 coup → micro-démo 2 coups”, jamais solution complète par défaut.
- **Montessori** : erreur = signal doux + auto-correction, sans punition, sans “Perdu”.
- **Non-compétitif** : pas de leaderboard, comparaison uniquement à soi.
